OBJETIVO: Sistema web de controle de fechamento mensal de departamento pessoal — 11 analistas, ~403 empresas, 12 atividades por competencia, substituindo planilhas Excel.

STACK: Supabase (PostgreSQL + Auth + RLS) . HTML/JS vanilla . Netlify . SheetJS (XLSX) . SVG nativo para graficos

DECISOES:
1. Single-file app (index.html) sem framework, sem build step
2. Perfis: admin, gestor, analista
3. 12 atividades fixas com regras NT automaticas: Filial sem mov = tudo NT; Matriz sem mov = so eSocial+DCTF ativos; sem adiantamento = grupo adiantamento NT; Filial = DARF+eSocial+DCTF sempre NT; Domestica = 7 atividades sempre NT (apontamento, previa, ok+cliente, fgts, provisoes, sindic, dctf)
4. empresa_competencia como snapshot mensal: tem_movimento, tem_adiantamento, funcionarios, analista_id — fallback para empresas.* quando NULL
5. analista_id em empresa_competencia e fonte de verdade para historico de carteiras — empresas.analista_id e o padrao para futuras competencias
6. Vigencia por data_inicio/data_fim em empresas — filtrada em todas as queries de competencia (data_inicio <= compLastDay AND data_fim >= compFirstDay)
7. Exclusao logica de empresas: status=encerrado + motivo (rescisao/encerramento) + data_fim
8. Competencias concluidas permanecem acessiveis e editaveis — reversao automatica para aberta se entrega removida
9. APP_NAME = "DP+ Suite" centralizado — renderBrand() injeta em login e header
10. Exportacao sempre .xlsx via SheetJS
11. Upsert de empresa_competencia sempre com onConflict: empresa_id,competencia_id
12. MODAL_ORIGIN, IMPORT_MODE, F_CART_*, SELECTED_EMPS resetados em boot() a cada login
13. Enum de tipo de empresa chama-se tipo_empresa_t (nao tipo_empresa) — valor 'Domestica' adicionado via ALTER TYPE tipo_empresa_t ADD VALUE

ENTREGUES:
- Login/logout Supabase Auth, boot por perfil
- Analista: checklist por competencia, filtros adiantamento/movimento, sumbar com vidas, chips de data editaveis, status Nao Ocorreu
- Gestor: dashboard 4 KPIs + drill-down por analista, aba Carteiras com filtros e edicao
- Admin: 6 abas — Competencias, Empresas, Carteiras, Importacao, Painel Executivo, Visao Geral
- Carteiras admin/gestor: tabela 12 atividades, chips editaveis, filtros (analista/mov/adiant), busca, exportacao Excel, bulk bar
- Atribuicao em lote: multi-select + modal, atualiza empresas.analista_id (permanente) + empresa_competencia.analista_id (historico comp atual)
- Painel Executivo: 8 KPIs com vigencia, producao, rankings empresas+vidas por analista historico, grafico SVG evolucao departamento
- Importacao 3 modos: Empresas (upsert por codigo), Funcionarios por Competencia, Responsaveis por Competencia
- Exportacao Excel: empresas e carteiras (.xlsx com 12 atividades + status)
- Competencias: abertura, exclusao (RPC SECURITY DEFINER), seletor com badge concluida, reversao automatica de status
- renderAnalista busca empresas via ec.analista_id para competencia atual, fallback empresas.analista_id
- renderCarteiras usa _analista_id (ec.analista_id ?? emp.analista_id) para display e filtro client-side
- renderExec usa analOf(e) = ecAnalIdx[e.id] ?? e.analista_id para perAnalyst
- _reRender() roteia para view correta apos saveDate/clearDate/saveNaoOcorreu via MODAL_ORIGIN
- checkCompConcluida() promove E reverte status automaticamente
- clearDate() chama checkCompConcluida() para permitir reversao
- Tipo Domestica: enum tipo_empresa_t atualizado, modal cadastro, importacao (aceita com/sem acento) e abrir_competencia v5 com 7 atividades NT
- migration_fase4.sql: constraint UNIQUE(empresa_id, competencia_id) + abrir_competencia v3
- migration_fase5.sql: coluna analista_id em empresa_competencia + backfill + abrir_competencia v4 + RLS entregas historico
- migration_fase6.sql (funcao apenas, sem ALTER TYPE): abrir_competencia v5 com regra Domestica
